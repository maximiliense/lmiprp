L‚Äôobjectif de cette s√©quence va √™tre de construire des m√©thodes et des algorithmes permettant de trouver des minimum locaux et/ou globaux de fonctions. Soit \(f:\mathbb{R}^d\mapsto\mathbb{R}\), nous souhaitons r√©soudre (ou du moins sur une partie du domaine de d√©finition de la fonction \(f\)) le probl√®me suivant :

\[x^\star=\text{argmin}_{x\in\mathbb{R}^d}f(x).\]

Dit autrement, nous consid√©rons \(f:\mathbb{R}^2\mapsto\mathbb{R}\) d√©finie par \(f(x, y)=\sqrt{(x^2+y-11)^2+(x+y^2-7)^2}\). \(f\) est continue et infiniment d√©rivable.

Nous avons en particulier les d√©riv√©es partielles suivantes :

\begin{equation} \frac{\partial f}{\partial x}(x, y)=\frac{2x^3+x(2y-21)+y^2-7}{\sqrt{(x^2+y-11)^2+(x+y^2-7)^2}} \end{equation}

et

\begin{equation} \frac{\partial f}{\partial y}(x, y)=\frac{x^2+2xy+2y^3-13y-11}{\sqrt{(x^2+y-11)^2+(x+y^2-7)^2}}. \end{equation}

Sans hypoth√®se sur la fonction \(f\), celle-ci peut √™tre tr√®s difficile √† minimiser. Soit \(x^{(0)}\in\mathbb{R}^2\), un algorithme permettant de chercher un minimum local en partant de \(x^{(0)}\) est la descente de gradient. Ce dernier suppose que nous avons acc√®s aux informations du premier ordre : le gradient \(\nabla f(x, y)\). Rappelons que le gradient est le vecteur construit √† partir des d√©riv√©es partielles \(\nabla f (x, y) = [\partial f(x,y)/\partial x, \partial f(x, y)/\partial y]^T\). Ce dernier donne le sens de la plus forte croissance de la fonction \(f\). Son oppos√© donne la plus forte pente. L‚Äôid√©e de l‚Äôalgorithme de descente de gradient est de suivre la direction donn√©e par ce dernier par petits pas. On note \(\boldsymbol{x}=(x,y)\). Nous avons ainsi :

\[\boldsymbol{x}^{(t+1)}=\boldsymbol{x}^{(t)}-\eta\nabla f(\boldsymbol{x}^{(t)})\]

o√π \(\eta>0\) est justement un param√®tre permettant de contr√¥ler la taille du pas.

class GradientDescent(object):
    def optimize(self, learning_rate = 0.1, nb_iterations=15, beta=None):
        # beta est notre variable ! 
        # si elle n'est pas fix√©e on la tire au hasard
        if beta is None:

        param_trace = [beta]
        loss_trace = [f(beta)]
        
        for i in range(nb_iterations):
            ####### Complete this part ######## or die ####################
            beta = beta - learning_rate * grad(beta)
            ###############################################################
            param_trace.append(beta)
            loss_trace.append(f(beta))
            
        return np.array(param_trace), np.array(loss_trace)

II. La descente de gradient √† pas optimal¬∂

Dans le sc√©nario pr√©c√©dent, nous avons du fixer un pas d‚Äôoptimisation \(\eta\) arbitraire. Ce dernier doit √™tre suffisament petit pour garantir que l‚Äôalgorithme converge et suffisamment grand pour que l‚Äôoptimisation se fasse. Il est possible de d√©finir une notion de pas d‚Äôoptimisation optimal. Cependant, celle-ci est souvent intractable en pratique (trouver le pas est plus couteux que l‚Äôoptimisation initiale). Dans certains cas, nous pouvons n√©anmoins le d√©terminer. C‚Äôest ce que nous allons faire ici. Consid√©rons maintenant la fonction suivante :

A = np.array([[1, 0], [0, 2]])
b = np.array([[2], [1]])

def f(x):
    if type(x) is np.ndarray:
        x = x.tolist()
    return (np.dot(np.dot(A, x).T, x)*0.5+np.dot(b.T, x))[0]

plot_loss_contour(f, three_dim=False, starting=[0, 2], 
                  title='2D heatmap of the function we want to optimize')
plot_loss_contour(f, three_dim=True, starting=[4, 4], 
                  title='3D heatmap of the function we want to optimize')

L‚Äôalgorithme de descente de gradient nous permet d‚Äôavancer dans la bonne direction. Cependant, le choix du pas \(\eta\) peut nous sembler insuffisant.

Soit \(\boldsymbol{\nu}=[x, y]^T\), la direction d‚Äôoptimisation \(\boldsymbol{d}^{(t)}=-\nabla f(\boldsymbol{\nu}^{(t)})\) et la fonction \(\gamma:t\mapsto f(\boldsymbol{\nu}^{(t)}+t\boldsymbol{d}^{(t)})\). La valeur de \(t\) qui minimise la fonction \(\gamma\) est un pas optimal pour une minimisation dans la direction du gradient. Sans contrainte particuli√®re sur la fonction \(f\), \(\gamma\) pourrait admettre un certain nombre de points critiques de natures et de valeurs diff√©rentes.

Les points critiques sont les points d‚Äôannulation de la d√©riv√©e : \(\{t\in\mathbb{R}:\ \gamma^\prime(t)=0\}\). On obtient assez facilement la d√©riv√©e de la mani√®re suivante :

\[\gamma^\prime(t)=\frac{\partial f}{\partial x}(\boldsymbol{\nu}^{(t)}+t\boldsymbol{d}^{(t)})\frac{\partial f}{\partial x}(\boldsymbol{\nu}^{(t)})+\frac{\partial f}{\partial y}(\boldsymbol{\nu}^{(t)}+t\boldsymbol{d}^{(t)})\frac{\partial f}{\partial y}(\boldsymbol{\nu}^{(t)})\]

On remarque que r√©soudre cette √©quation est rapidement probl√©matique et n√©cessite l‚Äôutilisation d‚Äôun autre algorithme de descente de gradient. En r√©alit√©, il y a grossi√®rement deux possibilit√©s :

  1. On peut trouver une valeur \(t\) analytiquement et c‚Äôest le choix qu‚Äôon doit faire,

  2. Il n‚Äôest pas possible de calculer \(t\) et on doit le calculer num√©riquement. Cependant, si on doit le calculer num√©riquement, alors, il devient n√©cessaire de calculer le gradient √† chaque √©tape, et dans ce cas, pourquoi ne pas juste se d√©placer dans l‚Äôespace des param√®tres avec notre vecteur \(\boldsymbol{[x, y]}\) ce qui nous donnerait une meilleure direction dans l‚Äôespace des param√®tres‚Ä¶

Il se trouve que la fonction d√©finie ci-dessus est : \(f(\boldsymbol{x})=\frac{1}{2}\langle Ax, x\rangle+\langle b, x\rangle\) o√π :

\[\begin{split}A=\begin{bmatrix} 1& 0\\ 0& 2\end{bmatrix}\end{split}\]

est sym√©trique d√©finie positive et

\[\begin{split}b=\begin{bmatrix}2\\1\end{bmatrix}\end{split}\]

Notons

\[\begin{split}\begin{aligned} f(\boldsymbol{x}+t\boldsymbol{d})&=\frac{1}{2}\langle A(x+t\boldsymbol{d}), x+t\boldsymbol{d}\rangle+\langle b, x+t\boldsymbol{d}\rangle\\ &=\frac{1}{2}(\langle A\boldsymbol{x},\boldsymbol{x}\rangle+t\langle A\boldsymbol{d},\boldsymbol{x}\rangle+t\langle A\boldsymbol{x},\boldsymbol{d}\rangle+t^2\langle A\boldsymbol{d},\boldsymbol{d}\rangle)+\langle \boldsymbol{b},\boldsymbol{x}\rangle+t\langle \boldsymbol{b},\boldsymbol{d}\rangle\\ &=f(\boldsymbol{x})+\frac{1}{2} t^2\langle A\boldsymbol{d},\boldsymbol{d}\rangle+t\langle A\boldsymbol{x}+\boldsymbol{b},\boldsymbol{d}\rangle \end{aligned}\end{split}\]

Notons de plus que \(\partial f(\boldsymbol{x})/\partial \boldsymbol{x}=A\boldsymbol{x}+\boldsymbol{b}=-\boldsymbol{d}\). Nous avons donc :

\begin{equation} f(\boldsymbol{x}+t\boldsymbol{d})=f(\boldsymbol{x})+\frac{1}{2} t^2\langle A\boldsymbol{d},\boldsymbol{d}\rangle-t\langle \boldsymbol{d},\boldsymbol{d}\rangle=f(\boldsymbol{x})+\frac{1}{2} t^2\langle A\boldsymbol{d},\boldsymbol{d}\rangle-\left\lVert\boldsymbol{d}\right\lVert^2 t \end{equation}

La direction \(\boldsymbol{d}=-\nabla f(\boldsymbol{x})\) est celle qui indique la plus forte pente. La variable \(t\) recherch√©e indique la taille du pas que l‚Äôon souhaite faire. Pour cela, nous devons chercher les points critiques de la fonction \(\gamma(t)=f(\boldsymbol{x}+t\boldsymbol{d})\) qui sont donn√©s en recherchant les points d‚Äôannulation de la d√©riv√©e. De plus, \(A\) (la Hessienne) √©tant d√©finie positive, nous savons que ces points critiques seront des minimums. Nous avons donc :

\[\frac{\partial \gamma}{\partial t}(t)=t\langle A\boldsymbol{d}, \boldsymbol{d}\rangle - \left\lVert\boldsymbol{d}\right\lVert^2=0\]

Et le point critique est donn√© par :

\[t=\frac{\left\lVert\boldsymbol{d}\right\lVert^2}{\langle A\boldsymbol{d}, \boldsymbol{d}\rangle}\]

class OptimalStepGradientDescent(object):
    def optimize(self, learning_rate=0.1, nb_iterations=15, beta=None):
        if beta is None:
            beta = np.random.uniform(-2, 2, size=(2, 1))
        else:
            beta = beta.reshape((2, 1))
            
        beta2 = beta.copy()

        param_trace = [beta]
        param_trace2 = [beta2]
        loss_trace = [f(beta)]
        loss_trace2 = [f(beta2)]
        it = 0
        stop = False
        
        for i in range(nb_iterations):
            d = -grad(beta)
            if d[0, 0] == d[1, 0] == 0.:
                stop = True
            else:
                ####### Complete this part ######## or die ####################
                t = np.linalg.norm(d)**2/np.dot(np.dot(A, d).T, d)
                beta = beta + t[0, 0] * d
                ###############################################################
                
                param_trace.append(beta)
                loss_trace.append(f(beta))
            
            # cette partie du code permet de calculer le gradient classique
            # afin que nous puissions le comparer avec la descente de gradient
            # √† pas optimal.
            beta2 = beta2 - learning_rate * grad(beta2)
            param_trace2.append(beta2)
            loss_trace2.append(f(beta2))
            it += 1
        return (np.array(param_trace), np.array(loss_trace), 
                np.array(param_trace2), np.array(loss_trace2))
        
gd = OptimalStepGradientDescent()

III. La m√©thode de Newton¬∂

Introduction¬∂

L‚Äô√©l√©ment de base de la m√©thode de Newton est le d√©veloppement limit√© d‚Äôune fonction \(f\) en \(x_0\). Soit \(f\in C^n(\mathbb{R})\) une fonction \(n\) fois d√©rivable de d√©riv√©e \(n^{eme}\) continue de \(\mathbb{R}\) dans \(\mathbb{R}\), son d√©veloppement limit√© √† l‚Äôordre \(n\) est donn√© par la formule suivante :

\[f(x)=\sum_{i=1}^n \frac{f^{(n)}(x_0)}{n!}(x-x_0)^n+o\big((x-x_0)^n\big)\]

Ainsi, la tangente √† notre fonction au point \(x_0\), ou approximation lin√©aire de notre fonction en \(x_0\), est donn√©e par :

\[f(x)\approx f(x_0)+f^\prime(x_0)(x-x_0)\]

et l‚Äôapproximation √† l‚Äôordre \(2\) par :

\[f(x)\approx f(x_0)+f^\prime(x_0)(x-x_0)+\frac{f^{\prime\prime}(x_0)}{2}(x-x_0)^2\]

Ces id√©es se g√©n√©ralisent √† des fonctions \(f:\mathbb{R}^n\mapsto\mathbb{R}\) :

\[f(x)\approx f(x_0)+\langle \nabla f(x_0), x-x_0\rangle + \frac{1}{2} (x-x_0)^TH_f(x-x_0)\]

La m√©thode¬∂

Soit \(f:\mathbb{R}\mapsto\mathbb{R}\) une fonction deux fois d√©rivables de d√©riv√©e seconde continue, l‚Äôobjectif de la m√©thode de Newton est de minimiser \(f\) :

\[\min_{x\in\mathbb{R}}f(x)\]

Supposons de plus \(f\) strictement convexe. Cette minimisation est s√©quentielle est produit une suite d‚Äôit√©r√©s \(\{x_0, x_1, ..., x_k\}\) o√π \(x_0\) est notre point de d√©part. chaque it√©r√© se rapproche un peu plus du minimiseur recherche \(x^\star\).

√Ä chaque it√©r√©e, l‚Äôapproximation √† l‚Äôordre \(2\) de \(f\) est elle-m√™me une fonction (strictement) convexe et coercive. Elle admet donc un minimum que l‚Äôon peut trouver en annulant la d√©riv√©e :

\[f(x_k+t)\approx f(x_k)+f^\prime(x_k)t+\frac{f^{\prime\prime}(x_k)}{2}t^2\]

On a donc :

\[\frac{d}{dt}(f(x_k)+f^\prime(x_k)t+\frac{f^{\prime\prime}(x_k)}{2}t^2)=f^\prime(x_k)+f^{\prime\prime}(x_k)t=0\Leftrightarrow t=-\frac{f^\prime(x_k)}{f^{\prime\prime}(x_k)}\]

Ce qui nous permet de fixer l‚Äôit√©r√© suivant : \(x_{k+1}=x_k-f^\prime(x_k)/f^{\prime\prime}(x_k)\).

Cette m√©thode se g√©n√©ralise bien s√ªr √† des fonctions √† plusieurs variables comme nous allons le voir lors d‚Äôune autre section.

Consid√©rons la fonction suivante :

IV. La descente de coordonn√©es¬∂

La descente de coordonn√©es ou coordinate descent consiste √† optimiser une fonction multivari√©e variable par variable. Soit \(f:\mathbb{R}^d\mapsto\mathbb{R}\) et le probl√®me d‚Äôoptimisation suivant :

\[x^\star=\text{argmin}_{x\in\mathbb{R}^d}f(x).\]

Contrairement √† la descente de gradient classique, ici, lors d‚Äôune √©tape d‚Äôoptimisation, une unique variable est mise √† jour √† la fois :

\[x^{(t)}_i=\text{argmin}_{x\in\mathbb{R}}f(x^{(t)}_1,\ldots, x_{i-1}^{(t)}, x, x_{i+1}^{(t-1)}, \ldots, x_d^{(t-1)}).\]

Consid√©rons la fonction √† deux variables suivantes : \(f(x, y)=5x^2-6xy+5y^2\).

def f(x):
    return 5*np.power(x[0], 2)-6*x[0]*x[1]+5*np.power(x[1], 2)

plot_loss_contour(f, three_dim=False, starting=[0, 2], 
                  title='2D heatmap of the function we want to optimize')

Consid√©rons \(g_y(x)=f(x, y)\) comme une fonction de \(x\) uniquement (i.e. \(y\) est fix√©) et d√©rivons :

\[g_y^\prime(x)=10x-6y.\]

Ainsi, l‚Äôit√©r√© suivant pour la variable \(x\) s‚Äôobtient de la mani√®re suivante : \(x^{(t)}=x^{(t-1)}-\eta g_y^\prime(x^{(t-1)})\). Le m√™me raisonement s‚Äô√©tend de mani√®re totalement sym√©trique pour obtenir l‚Äôit√©r√© de la variable \(y\) (remarquez que \(f(x, y)= f(y, x)\)).

class CoordinateDescent(object):
    def optimize(self, x_start, nb_iterations=15, learning_rate=0.01):
        optimization_steps = [np.copy(x_start)]
        x_t = x_start
        
        for i in range(nb_iterations):
            ####### Complete this part ######## or die ####################
            x_t[0]=x_t[0]-learning_rate*(10*x_t[0]-6*x_t[1])
            ###############################################################

            optimization_steps.append(np.copy(x_t))
            
            ####### Complete this part ######## or die ####################
            x_t[1]=x_t[1]-learning_rate*(10*x_t[1]-6*x_t[0])
            ###############################################################
            
            optimization_steps.append(np.copy(x_t))

            
        return np.array(optimization_steps)
