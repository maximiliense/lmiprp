Sous-diff√©rentiel et le cas du Lasso ‚òïÔ∏è‚òïÔ∏è‚òïÔ∏è¬∂

Introduction¬∂

Soit \(X\in\mathbb{R}^{n\times d}\) et \(\boldsymbol{y}=\mathbb{R}^n\). De mani√®re assez directe, le probl√®me des moindres carr√©s se formule de la mani√®re suivante :

\[\beta^\star=\text{argmin}_{\beta\in\mathbb{R}^d}\lVert X\beta-\boldsymbol{y}\rVert_2^2.\]

Supposons que \(X^TX\) soit inversible. Alors, en annulant le gradient, on obtient :

\[\beta^\star=(X^TX)^{-1}X^T\boldsymbol{y}.\]

Si nous avions souhait√© minimiser notre fonction objectif via une descente de gradient (par exemple dans le cas o√π \(X^TX\) n‚Äôest pas inversible ou si le co√ªt de son inversion est trop important), alors le gradient √† utiliser est :

\[\nabla (\lVert X\beta-\boldsymbol{y}\rVert_2)=2X^TX-2X^t\boldsymbol{y}.\]

Cependant, comme nous l‚Äôavons vu dans une s√©quence pr√©c√©dente, il est parfois pr√©f√©rable de ‚Äúr√©gulariser‚Äù ce probl√®me d‚Äôoptimisation de mani√®re √† obtenir un vecteur \(\beta^\star\) g√©n√©ralisant mieux √† de nouveaux exemples d‚Äôapprentissage en machine learning ou offrant plus de stabilit√©. Le cas de la r√©gularisation \(\lVert\cdot\rVert_1\) dit \(\ell_1\), qu‚Äôon appelle Lasso, est int√©ressant car il permet de faire de la s√©lection de variables. En effet, le vecteur \(\beta^\star\) obtenu apr√®s minimisation est sparse.

Rappelons que la norme \(\ell_1\) se d√©finie de la mani√®re suivante :

\[\lVert x \rVert_1=\sum_i |x_i|.\]

Le probl√®me r√©gularis√© peut donc se reformuler de la mani√®re suivante :

\[\beta^\star=\text{argmin}_{\beta\in\mathbb{R}^d}\lVert X\beta-\boldsymbol{y}\rVert_2^2+\lambda \lVert \beta\rVert_1,\]

o√π \(\lambda\geq0\) contr√¥le la quantit√© de r√©gularisation.

Le probl√®me ici est que \(\lVert \beta\rVert_1\) n‚Äôest pas diff√©rentiable partout - la valeur absolu n‚Äôest pas d√©rivable en \(0\). On aurait pu se dire que ce n‚Äôest pas un probl√®me et que notre descente de gradient aurait √† coup s√ªr ‚Äúsaut√©‚Äù les points non-diff√©rentiables, mais ces derniers s‚Äôav√®rent justement √™tre solution de notre probl√®me : c‚Äôest le c√¥t√© sparse. Nous allons, dans cette s√©quence introduire quelques √©l√©ments permettant d‚Äôaborder ce probl√®me : le sous-diff√©rentiel.

√Ä la fin de cette s√©quence, nous d√©riverons un algorithme permettant d‚Äôobtenir la solution du probl√®me d‚Äôoptimisation Lasso !

I. Quelques bases¬∂

Nous allons ici introduire quelques √©l√©ments math√©matiques permettant de d√©river les algorithmes d‚Äôoptimisation voulus. Tout d‚Äôabord, soit \(f:\mathbb{R}\mapsto\mathbb{R}\) une fonction d√©rivable, alors sa tangente en \(a\) a pour √©quation :

\[t(x)=f(a)+f^\prime(a)(x-a).\]

Si \(f\) est convexe, alors \(f(y)\geq t(x),\ \forall x, y\in\mathbb{R}\). Rappelons que la convexit√© d‚Äôune fonction \(f\) nous indique que \(\forall x,y\in\mathbb{R},\ \lambda\in [0, 1]\), on a :

\[f(\lambda x+(1-\lambda)y)\leq \lambda f(x)+(1-\lambda)f(y).\]

La sous-d√©riv√©e¬∂

L‚Äôid√©e de la sous-d√©riv√©e est de pouvoir g√©n√©raliser l‚Äôid√©e que la d√©riv√©e contr√¥le la tangente dans le cas d‚Äôune fonction convexe. Soit \(f\) une fonction convexe, \(s\) est une sous d√©riv√©e de \(f\) en \(a\) si :

\[f(x) \geq f(a)+s(x-a),\ \forall x\in\mathbb{R}.\]

La sous-d√©riv√©e n‚Äôest pas forc√©ment unique en un point \(a\) et on appelle sous-diff√©rentiel l‚Äôensemble des sous-d√©riv√©s qu‚Äôon note \(\partial f(a)\). Cependant, si \(f\) est diff√©rentiable en \(a\), alors \(\partial f(a)=\{f^\prime(a)\}\). On observe assez rapidement que si \(f\) est concave au voisinage de \(a\), alors elle n‚Äôadmet pas de sous-d√©riv√©e : \(\partial f(x)=\emptyset\).

La figure suivante illustre quelques sous-d√©riv√©es pour les fonctions \(f(x)=|x|\) et \(g(x)=x^2\).

On observe bien l‚Äôinfinit√© des sous-d√©riv√©es de la valeur absolue en \(0\) et l‚Äôunicit√© en tout point pour la fonction \(x^2\).

Un certain nombre de r√®gles de calcul usuelles se g√©n√©ralisent assez bien √† ce nouveau concept. Soit \(f, g\) deux fonctions et un scalaire \(\alpha\), nous avons:

\[\partial (\alpha(f+g))=\alpha\partial f+\alpha \partial g.\]

Soit \(f\) une fonction d√©rivable et soit le probl√®me d‚Äôoptimisation suivant :

\[x^\star=\text{argmin}_{x\in\mathbb{R}}f(x).\]

La condition d‚Äôoptimalit√© du premier ordre, dit de Fermat, nous donne l‚Äô√©quivalence suivante :

\[x^\star\text{ est un minimum local}\Leftrightarrow f^\prime(x^\star)=0.\]

Si \(f\) est convexe, alors \(x^\star\) est un minimum global. Qu‚Äôen est-il dans le cas o√π \(f\) n‚Äôest pas d√©rivable en \(x^\star\) mais sous-diff√©rentiable ? Supposons \(f\) convexe. On a :

\[x^\star\text{ est un minimum global}\Leftrightarrow 0\in\partial f(x^\star).\]

Lorsqu‚Äôon consid√®re des fonctions de \(\mathbb{R}^n\) dans \(\mathbb{R}\), alors les principes pr√©c√©dents se g√©n√©ralisent. On parle alors de sous-gradient.

II. L‚Äôalgorithme d‚Äôoptimisation proximal et son application au Lasso¬∂

Reprenons le probl√®me d‚Äôoptimisation r√©gularis√© suivant :

\[\beta^\star=\text{argmin}_{\beta\in\mathbb{R}^d}\lVert X\beta-\boldsymbol{y}\rVert_2+\lambda \lVert \beta\rVert_1 = f(\beta)+g(\beta),\]

o√π \(f\) et \(g\) sont des fonctions convexes, \(f\) une fonction diff√©rentiable.

Une strat√©gie d‚Äôoptimisation de ce probl√®me est ce qu‚Äôon appelle l‚Äôalgorithme proximal. Ce dernier d√©coupe chaque pas d‚Äôoptimisation en deux √©tapes. La premi√®re consiste √† optimiser \(f\) en suivant son gradient (comme dans la descente de gradient). La seconde √©tape consiste √† ‚Äúoptimiser \(g\) dans le voisinage du point pr√©c√©demment obtenu‚Äù :

\[\begin{split}\text{pas d'optimisation : }\begin{cases}u^{(t+1)}&=\beta^{(t)}-\gamma\nabla f(\beta^{(t)})\\ \beta^{(t+1)}&=\textbf{prox}_{\gamma g}(u^{(t+1)})\end{cases}\end{split}\]

Cette algorithme s‚Äôappuie sur un op√©rateur qu‚Äôon appelle ‚Äúl‚Äôop√©rateur proximal‚Äù :

\[\textbf{prox}_{\gamma g}(u)=\text{argmin}_y\big\{g(y)+\frac{1}{2\gamma}\lVert y-u\rVert_2^2\big\}.\]

On cherche √† minimiser \(g\) depuis un point \(u\) en gardant la distance \(\ell_2\) au carr√© faible.

La p√©nalit√© \(\ell_1\) du Lasso et l‚Äôop√©rateur proximal : Soit \(g(x)=\lambda \lVert x\rVert_1\). On cherche donc √† trouver un point d‚Äôannulation de la sous-diff√©rentielle de la fonction objectif de notre op√©rateur proximal. On a donc :

\[\textbf{prox}_{\gamma g}(u)=\textbf{prox}_{\gamma \lambda \lVert \cdot\rVert_1}(u),\]

et en calculant la diff√©rentielle, nous obtenons :

\[\partial(\lVert y\rVert_1+\frac{1}{2\lambda\gamma}\lVert y-u\rVert_2^2)=\partial \lVert y\rVert_1+\frac{1}{2\lambda\gamma}\partial \lVert y-u \rVert_2^2=\partial \lVert y\rVert_1+\Big\{\frac{y-u}{\lambda\gamma}\Big\}.\]

Le probl√®me est s√©parable et peut √™tre consid√©r√© coordonn√©e par coordonn√©e :

\[\begin{split}(\partial \lVert y\rVert_1)_i=\begin{cases}1&\text{ si }y_i>0\\ [-1, 1]&\text{ si }y_i=0\\ -1&\text{ sinon.}\end{cases}\end{split}\]

En combinant les deux sous-diff√©rentielles, et en traitant les deux probl√®mes coordonn√©e par coordonn√©e, nous obtenons :

\[\begin{split}\partial(\lVert y\rVert_1+\frac{1}{2\lambda\gamma}\lVert y-u\rVert_2^2)=\begin{cases}1+\frac{y_i-u_i}{\lambda\gamma}&\text{ si }y_i>0\\ \Big[-1+\frac{y_i-u_i}{\lambda\gamma}, 1+\frac{y_i-u_i}{\lambda\gamma}\Big]&\text{ si }y_i=0\\ -1+\frac{y_i-u_i}{\lambda\gamma}&\text{ si }y<0.\end{cases}\end{split}\]

Notre objectif est de trouver \(y_i\) tel que \(0\in\partial(\lVert y\rVert_1+\frac{1}{2\lambda\gamma}\lVert y-u\rVert_2^2)\). Trois sc√©narios sont possibles. Tout d‚Äôabord :

\[0=1+\frac{y_i-u_i}{\lambda\gamma}\text{ et }y_i>0\Leftrightarrow y_i=u_i-\lambda\gamma\text{ et }u_i>\lambda\gamma.\]

Les deux autres sc√©narios se construisent exactement de la m√™me mani√®re. On obtient au final :

\[\begin{split}\textbf{prox}_{\gamma \lambda \lVert \cdot\rVert_1}(u)_i=\begin{cases}u_i-\lambda\gamma&\text{ si }u_i>\lambda\gamma\\ u_i+\lambda\gamma&\text{ si } u_i<-\lambda\gamma\\ 0&\text{ si }u_i\in[-\lambda\gamma,\lambda\gamma].\end{cases}\end{split}\]

La fonction objectif non r√©gularis√©e du Lasso : Soit \(f(\beta)=\lVert X\beta-y\rVert_2^2\). On a comme nous l‚Äôavons d√©j√† vu plusieurs fois :

\[\nabla f(\beta)=(X^TX)\beta-X^Ty.\]

L‚Äôalgorithme proximal appliqu√© au Lasso : Soit le probl√®me Lasso suivant :

\[\beta^\star=\text{argmin}_{\beta\in\mathbb{R}^d}\lVert X\beta-\boldsymbol{y}\rVert_2+\lambda \lVert \beta\rVert_1 = f(\beta)+g(\beta).\]

L‚Äôalgorithme proximal est ainsi un algorithme d‚Äôoptimisation qui r√©alise une suite de pas d‚Äôoptimisation. Un pas d‚Äôoptimisation est construit de la mani√®re suivante :

\[\begin{split}\text{pas d'optimisation : }\begin{cases}u^{(t+1)}&=\beta^{(t)}-\gamma\nabla f(\beta^{(t)})\\ \beta^{(t+1)}&=\textbf{prox}_{\gamma \lambda \lVert \cdot\rVert_1}(u^{(t+1)})\end{cases}\end{split}\]

class ProximalOptimization(object):
    def __init__(self, X, y):
        self.X = X
        self.y = y
        
    def gradient(self, beta):
        ####### Complete this part ######## or die ####################
        return (np.dot(np.dot(self.X.T, self.X), beta)-np.dot(self.X.T, self.y))/self.X.shape[0]
        ###############################################################
    
    def gradient_descent_step(self, learning_rate, beta):
        ####### Complete this part ######## or die ####################
        g = self.gradient(beta)
        return beta-learning_rate*g
        ###############################################################
    
    def proxy_step(self, learning_rate, lambda_, u):
        ####### Complete this part ######## or die ####################
        gamma = lambda_*learning_rate
        to_zero = np.abs(u) < gamma
        u[to_zero] = 0
        neg = u < -gamma
        u[neg] = u[neg] + gamma
        pos = u > gamma
        u[pos] = u[pos] - gamma
        return u
        ###############################################################
        
    def optimize(self, learning_rate, beta, lambda_, nb_iterations):
        ####### Complete this part ######## or die ####################
        param_trace = [np.copy(beta)]
        for it in range(nb_iterations):
            u = self.gradient_descent_step(learning_rate, beta)
            beta = self.proxy_step(learning_rate, lambda_, u)
            param_trace.append(np.copy(beta))
        return np.array(param_trace)
        ###############################################################
