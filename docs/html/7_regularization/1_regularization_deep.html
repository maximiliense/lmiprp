Les techniques de r√©gularisation sont fondamentales en machine learning. Elles nous permettent de consid√©rer des classes de fonctions complexes tout en favorisant certaines solutions dont on sait qu‚Äôelles auront tendances √† √™tre plus stables et √† mieux g√©n√©raliser √† de nouvelles donn√©es. Les r√©gularisations peuvent √™tre explicites comme au travers de l‚Äôusage d‚Äôune p√©nalit√© \(\ell_1\) ou \(\ell_2\) comme nous avons pu le voir. Elles peuvent √™tres implicites en r√©sultant par exemple d‚Äôun effet de bord de l‚Äôoptimiseur comme nous le verrons plus bas.

Le deep learning offre tout un ensemble de strat√©gies qu‚Äôon ne retrouve pas syst√©matiquement ailleurs. Cela est d√ª √† la structure m√™me de ces fonctions mais aussi au types de t√¢ches et de donn√©es qu‚Äôelles permettent de traiter avec succ√®s. Cette s√©quence est donc consacr√©e aux m√©thodes de r√©gularisations utilis√©es en deep learning.

Toute cette s√©quence consid√©rera \(\mathcal{X}\) comme √©tant l‚Äôespace des donn√©es d‚Äôentr√©e, \(\mathcal{Y}\) celui de nos labels et \(S_n=\{(x_i, y_i)\}_{i\leq n}\) notre jeu de donn√©es d‚Äôapprentissage et \(T_n=\{(x_i, y_i)\}_{i\leq m}\) notre jeu de test.

I. R√©gularisation de la norme du vecteur de param√®tres¬∂

De la m√™me mani√®re que pour d‚Äôautres approches de machine learning, les r√©seaux de neurones en √©tant des familles param√©triques de fonctions peuvent √™tre p√©nalis√©s au travers de l‚Äôapprentissage. Notons \(L(\theta, S_n)\) la loss √† optimiser sur notre jeu de donn√©es \(S_n\) o√π \(\theta\) est une param√©trisation donn√©e de notre mod√®le. Nous d√©finissons notre loss r√©gularis√©e de mani√®re tr√®s standard de la mani√®re suivante : $\(\tilde{L}(\theta, S_n)=L(\theta, S_n)+\lambda P(\theta),\)\( o√π \)\lambda > 0$ contre la quantit√© de r√©gularisation souhait√©e.

A. La r√©gularisation \(\ell_2\), weight decay ou encore Ridge¬∂

La r√©gularisation \(\ell_2\) est peut-√™tre la r√©gularisation explicite la plus transversalement adopt√©e. Dans ce cas de figure la p√©nalit√© n‚Äôest autre que la norme \(\ell_2\) de notre vecteur de param√®tres : $\(P(\theta)=\frac{1}{2}\lVert\theta\rVert_2^2,\)\( o√π le \)1/2$ n‚Äôa aucun int√©r√™t autre que calculatoire. Une s√©quence compl√®te sera d√©di√©e √† cette r√©gularisation. Pour l‚Äôinstant, observons intuitivement l‚Äôeffet de cette derni√®re. \begin{equation} \begin{aligned} \nabla \tilde{L}(\theta, S_n)&=\nabla L(\theta, S_n)+ \nabla\frac{\lambda}{2}\lVert\theta\rVert_2^2\ &=\nabla L(\theta, S_n)+\lambda \theta \end{aligned} \end{equation}

Rappelons nous que le pas de descente de gradient est donn√© par : $\(\theta^{t+1}=\theta^t-\eta\nabla \tilde{L}(\theta^t)\)\( o√π \)\eta>0\( est le pas d'apprentissage. On obtient donc dans le cas d'une p√©nalit√© \)\ell_2\( le pas d'optimisation suivant : \)\(\theta^{t+1}=(1-\eta\lambda)\theta^t-\eta\nabla L(\theta^t, S_n), \)$

o√π on observe qu‚Äôavant m√™me de se d√©placer dans la direction de la plus forte pente (\(-\nabla\)), notre vecteur de param√®tre se voit contract√© par la p√©nalit√© \(\lambda\).

La p√©nalit√© \(\ell_2\) √©tant standard √† beaucoup de mod√®les, n‚Äôh√©sitez pas √† vous r√©f√©rer aux autres s√©quences afin d‚Äôapprofondir le sujet

class Net(nn.Module):
    def __init__(self):
        super(Net, self).__init__()
        self.conv1 = nn.Conv2d(3, 16, 5)
        self.pool = nn.MaxPool2d(2, 2)
        self.conv2 = nn.Conv2d(16, 32, 5)
        self.fc1 = nn.Linear(32 * 5 * 5, 120)
        self.fc2 = nn.Linear(120, 84)
        self.fc = nn.Linear(84, 10)

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x)))
        x = self.pool(F.relu(self.conv2(x)))
        x = x.view(-1, 32 * 5 * 5)
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = self.fc(x)
        return x

criterion = nn.CrossEntropyLoss()

def train(model, criterion, optimizer, n_epoch=2, eval_frequency=2):
    loss_history = []
    valid_loss_history = []

    acc_history = []
    val_acc_history = []

    for epoch in range(n_epoch):  # loop over the dataset multiple times
        print("[EPOCH %d, LR: %s]"%(epoch + 1, str(0.01)))

        #iterate over the training batches until all all samples have been considered
        model.train()
        running_loss = 0.0
        for i, data in enumerate(trainloader, 0):
            if i % 5 == 4:
                print('\r[Batch id: %d/%d]' % (i+1, len(trainloader)), end='')
            # get the inputs; data is a list of [inputs, labels]
            inputs, labels = data

            # Convert to cuda is device is gpu
            # inputs = inputs.cuda()
            # labels = labels.cuda()

            # Zero the parameter gradients
            optimizer.zero_grad()

            # Forward : compute outputs for all layers 
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            running_loss += loss.item()

            # Backward: Compute gradient wrt parameters of all layers
            loss.backward()


            optimizer.step()

        print(' train loss: %.3f' % (running_loss/len(trainloader)))


        if (epoch + 1) % eval_frequency == 0:
            #Compute training statistics
            accuracy_train = test(model, trainloader, 'train')
            accuracy_test = test(model, testloader, 'test')
            loss_train = loss_eval(model, trainloader, 'train')
            loss_test = loss_eval(model, testloader, 'test')
            loss_history.append(loss_train)
            acc_history.append(accuracy_train)
            valid_loss_history.append(loss_test)
            val_acc_history.append(accuracy_test)

        print("\n")

    print('**** Finished Training ****')
    return loss_history, val_acc_history

def test(model, loader, title):
    model.eval()  # on passe le modele en mode evaluation
    correct = 0
    total = 0
    with torch.no_grad():
        for data in loader:
            images, labels = data
            # images = images.cuda()
            # labels = labels.cuda()
            outputs = model(images)
            predicted = torch.nn.functional.softmax(outputs, dim=1).max(dim=1)
            
            total += labels.size(0)
            correct += (predicted[1] == labels).sum().item()
            
    model.train()  # on remet le modele en mode apprentissage
    print('Accuracy du modele sur le jeu de ', title, ': %.2f' % (correct / total))
    return correct / total
  
def loss_eval(model, loader, title):
    model.eval()  # on passe le modele en mode evaluation
    running_loss = 0.
    total = 0.
    with torch.no_grad():
        for data in loader:
            images, labels = data
            # images = images.cuda()
            # labels = labels.cuda()
            outputs = model(images)

            total += 1.
            running_loss += criterion(outputs, labels).item()

    model.train()  # on remet le modele en mode apprentissage
    print('Loss du modele sur le jeu de ', title, ': %.2f' % (running_loss / total))
    return (running_loss / total)

IV. Le dropout¬∂

L‚Äôid√©e du dropout est de perturber l‚Äôapprentissage en mettant al√©atoirement certains neurones de notre r√©seau √† \(0\). Ainsi, le mod√®le, afin de minimiser l‚Äôerreur, est oblig√© d‚Äôexploiter de l‚Äôinformation provenant de plusieurs neurones : cette redondance limite l‚Äôeffet de surapprentissage tr√®s li√© √† la capacit√© du mod√®le √† exploiter le bruit des donn√©es, bruit qui par d√©finition n‚Äôoffre pas de redondance.

Notons \(\sigma\) une non-lin√©arit√© et \(w_{ij}, b_{ij}\) les param√®tres de la couche \(i\) et du neurone \(j\) de notre r√©seau de neurones. Nous avons donc pour la couche \(i\) et son neurone \(j\) : $\(\sigma(\langle w_{ij}, z\rangle +b_{ij}),\)\( o√π \)z\( est la sortie de la couche pr√©c√©dente. L'id√©e du *Dropout* est de consid√©rer un vecteur \)\delta\( tel que \)\delta_i\sim \text{Bernoulli}(1-p)\( o√π \)p\( est le param√®tre de *dropout* si \)p=0\( alors le *dropout* est d√©sactiv√©, et si \)p\( est proche de \)1\( alors aucune information ne passe. Une couche de dropout est : \)\(\text{do}(z)=\frac{\delta \odot z}{1-p}, \)\( o√π \)\odot\( est la multiplication √©l√©ment par √©l√©ment. Le d√©nominateur permet de garantir que "l'amplitude" du vecteur sortant du *dropout* est la m√™me avec ou sans *dropout*. Plus rigoureusement, nous avons : \)\(\mathbb{E}\Big[\frac{\delta_i}{1-p}\Big]=1.\)\( Le dropout est d√©sactiv√© lorsqu'on utilise le mod√®le pour faire des pr√©dictions (et sa normalisation √©galement). En reprenant notre couche \)i\(, tous ses neurones \)j$ ont la forme suivante :

\[\sigma(\langle w_{ij}, \text{do}(z)\rangle +b_{ij}),\]

V. La batch normalization¬∂

La batch normalization est une autre technique de r√©gularisation qui n‚Äôa cependant, √† premi√®re vue, rien √† voir avec la r√©gularisation. Ses effets sont encore mal compris et sont probablement li√©s √† la r√©gularisation implicite qui d√©pend de l‚Äôoptimiseur lui-m√™me (comme nous allons le voir plus bas). La batch normalization ne fonctionne que si les donn√©es arrivent par batch de taille sup√©rieure √† 1. Nous allons voir une mani√®re standard de faire de la batch normalization mais il est √† noter que la couche est g√©n√©ralement ultra-configurable et beaucoup de ses composantes peuvent √™tre d√©sactiv√©es. Une couche est d√©crite par l‚Äô√©quation suivante :

\[\text{bn}(z)=\frac{z-\mathbb{E}\big[z\big]}{\sqrt{\text{Var}(z)+\epsilon}}\odot \gamma+\beta,\]

o√π \(z\) est un batch de donn√©es, \(\gamma\) et \(\beta\) sont des vecteurs dont la dimension est celle d‚Äôune donn√©e de \(z\) et dont les op√©rations sont appliqu√©es √©l√©ment par √©l√©ment dans \(z\). L‚Äôesp√©rance et la variance sont calcul√©es √† partir du batch mais peuvent conserver une inertie des batchs pr√©c√©dents. Lorsque notre mod√®le passe en mode pr√©diction, on r√©utilise les statistiques de moyenne et de variance obtenues lors de l‚Äôapprentissage.

Attention, certains objets \(\texttt{pytorch}\) demande si leur entr√©e sera un vecteur (1d), une image (2d), etc. C‚Äôest le cas de la batch normalization avec notamment BatchNorm1d ou BatchNorm2d, etc.

def fullyconnected_layer(in_f, out_f):
    """
    " this function returns a fully connected layer with batchnorm
    """
    return nn.Sequential(
        nn.Linear(in_f, out_f),
        nn.BatchNorm1d(out_f),
        nn.ReLU()
    )

class FullyConnectedNetwork(nn.Module):
    def __init__(self, n_labels=2, n_input=2, architecture=(2,)):
        """
        " n_labels is the dimension of the output
        " n_input is the dimension of the input
        " architecture describes the series of hidden layers
        """
        super(FullyConnectedNetwork, self).__init__()

        layer_size = [n_input] + [i for i in architecture]

        layers = [
            fullyconnected_layer(in_f, out_f) for in_f, out_f in zip(layer_size, layer_size[1:])
        ]
        self.layers = nn.Sequential(*layers)
        self.fc = nn.Linear(architecture[-1], n_labels)

    def forward(self, x):
        x = self.layers(x)
        x = self.fc(x)
        return x
