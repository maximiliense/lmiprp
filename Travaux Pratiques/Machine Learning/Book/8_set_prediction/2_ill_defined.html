Jeu d‚Äôapprentissage uniquement multi-classes ‚òïÔ∏è¬∂

Objectifs de la s√©quence

  • √ätre sensibilis√©¬†:

    • √† l‚Äôid√©e de pr√©diction d‚Äôensembles.

  • √ätre capable¬†:

    • de formaliser la pr√©diction \textit{average-K},

    • d‚Äôimpl√©menter un \textit{average-K} avec \(\texttt{pytorch}\).

Dans la plupart des donn√©es et y-compris sur des probl√®mes clairement ‚Äúensemblistes‚Äù, nous ne disposons g√©n√©ralement QUE d‚Äôune seule classe par exemple d‚Äôapprentissage. Le probl√®me n‚Äôest pas tant qu‚Äôil n‚Äôy en ait qu‚Äôune mais que nous n‚Äôavons AUCUN exemple d‚Äôabsence. Ainsi, nous pouvons construire un probl√®me d‚Äôoptimisation qui r√©compense les pr√©dictions d‚Äôensembles contenant la bonne r√©ponse mais qui ne ferait rien dans le cas contraire. Le probl√®me est que finalement, en faisant cela, il suffit que notre mod√®le retourne toujours toutes les classes pour minimiser notre loss. Hors, ce n‚Äôest pas ce que nous souhaitons.

Soit \(\mathcal{X}\) notre espace d‚Äôentr√©e, \(\mathcal{Y}=\{1, \ldots, C\}\) celui des pr√©dictions et notons \(S_n=\{(X_i, Y_i)\}_{i\leq n}\) notre jeu de donn√©es sur \(\mathcal{X}\times\mathcal{Y}\). Nous traitons bien s√ªr ici d‚Äôun probl√®me de classification (cf. \(\mathcal{Y}\)). Comme c‚Äôest presque tout le temps le cas, chaque exemple d‚Äôapprentissage est associ√© √† un unique label et nous souhaiterions construire un mod√®le capable de pr√©dire un ensemble plut√¥t qu‚Äôun seul √©l√©ment. Nous cherchons ainsi des applications :

\[h:\mathcal{X}\rightarrow\mathcal{P}(\mathcal{Y}).\]

Cette s√©quence se concentrera sur deux strat√©gies : les pr√©dictions Top-K et les pr√©dictions Average-K. Nous formaliserons le probl√®me d‚Äôoptimisation qu‚Äôelles cherchent √† r√©soudre, nous en d√©duirons le classifieur de Bayes et nous illustrerons ces strat√©gies via des exemples.

I. Pr√©diction top-K¬∂

A. Une introduction th√©orique¬∂

L‚Äôapproche la plus na√Øve consiste √† se dire que s‚Äôil y a de l‚Äôambigu√Øt√© dans nos donn√©es, alors il de ne pas pr√©dire qu‚Äôune seule classe mais \(K\) classes √† chaque fois. Le probl√®me n‚Äôest pas n√©cessairement √©vident et nous situerons donc dans un cadre particulier o√π notre mod√®le est un estimateur de la probabilit√© conditionnelle de chaque classe :

\[\hat{\eta}_k(x)=\mathbb{P}(Y=k|X=x),\]

sachant que par d√©finition de la notion de probabilit√©, nous avons bien s√ªr :

\[\sum_k \hat{\eta}_k(x)=1.\]

Notre objectif est donc de construire un classifieur \(\mathcal{S}:\mathcal{X}\rightarrow\mathcal{P}(\mathcal{Y})\) tel que \(|\mathcal{S}(x)|=K,\ \forall x\in\mathcal{X}\). Nous cherchons ici √† minimiser l‚Äôerreur suivante :

\[R(\mathcal{S})=\mathbb{P}(Y\not\in \mathcal{S}(X)).\]

Question

√Ä chaque pr√©diction exactement \(K\) classes sont retourn√©es. Cette fois-ci, nos pr√©dictions se font dans l‚Äôespace \(\mathcal{P}(\mathcal{Y})\) l‚Äôensemble des parties de \(\mathcal{Y}\). Notons \(\eta_k(x)=\mathbb{P}(Y=k|X=x)\). Trouvez le classifieur de Bayes

B. En pratique¬∂

En pratique, on peut chercher √† optimiser une loss directement d√©di√©e au Top-K ou, √† l‚Äôinverse, optimiser un mod√®le avec une loss classique permettant d‚Äôavoir un estimateur \(\hat{\eta}\) et ensuite d‚Äôutiliser la r√®gle de classification sur cet estimateur. C‚Äôest la seconde approche que nous choisissons ici.

Notre estimateur de la probabilit√© conditionnelle \(\hat{\eta}\) est tout simplement notre r√©seau de neurones. Notez cependant que les sorties ne sont pas normalis√©es car, pour des raisons de stabilit√©, nous pr√©f√©rons laisser le soin de la normalisation √† la loss cross-entropy. import torch.nn as nn
import torch.nn.functional as F
from torchvision import models, transforms

class Net(nn.Module):
    def __init__(self):
        super(Net, self).__init__()
        self.conv1 = nn.Conv2d(3, 16, 5)
        self.pool = nn.MaxPool2d(2, 2)
        self.conv2 = nn.Conv2d(16, 32, 5)
        self.fc1 = nn.Linear(32 * 5 * 5, 120)
        self.fc2 = nn.Linear(120, 84)
        self.fc = nn.Linear(84, 10)

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x)))
        x = self.pool(F.relu(self.conv2(x)))
        x = x.view(-1, 32 * 5 * 5)
        x = F.relu(self.fc1(x))
        x = F.relu(self.fc2(x))
        x = self.fc(x)
        return x

model = Net()

import torch.optim as optim
from torch.optim.lr_scheduler import MultiStepLR

#Choose the loss function
criterion = nn.CrossEntropyLoss()

#Optimizer
optimizer = optim.SGD(model.parameters(), lr=0.05, momentum=0.9, weight_decay=0.)

import matplotlib.pyplot as plt

loss_history = []
for epoch in range(2):  # loop over the dataset multiple times

    running_loss = 0.0
    for i, data in enumerate(trainloader, 0):
        # get the inputs; data is a list of [inputs, labels]
        inputs, labels = data

        # zero the parameter gradients
        optimizer.zero_grad()

        # forward + backward + optimize
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        # print statistics
        running_loss += loss.item()

print('\r**** Finished Training ****')
plt.figure(figsize=(12, 8))
plt.plot([i for i in range(1, len(loss_history)+1)], loss_history,

II. Pr√©diction Average-K¬∂

A. Une introduction th√©orique¬∂

On sent cependant assez rapidement que l‚Äôapproche Top-K n‚Äôest pas totalement satisfaisante. On souhaiterait par exemple ne retourner qu‚Äôun seul √©l√©ment lorsque la classe de l‚Äôimage est √©vidente et simple et plus de \(K\) √©l√©ments lorsque l‚Äôimage est difficile √† identifier. Nous cherchons toujours ici √† minimiser l‚Äôerreur :

\[R(\mathcal{S})=\mathbb{P}(Y\not\in \mathcal{S}(X)).\]

La diff√©rence principale est la contrainte que l‚Äôon souhaite imposer sur notre mod√®le. Le classifieur de Bayes est ainsi la solution du probl√®me :

\[\mathcal{S}^\star \in \text{argmin}_{\mathcal{S}}\mathbb{P}\big(Y\not\in \mathcal{S}(X)\big),\ \text{s.t. }\mathbb{E}\big[|S(X)|\big]\leq K,\]

o√π la premi√®re appartenance vient du fait que le classifieur de Bayes n‚Äôest pas n√©cessairement unique. En reprenant la formulation avec p√©nalit√©, \(\exists\lambda > 0\) tel que :

\[\mathcal{S}^\star_\lambda \in \text{argmin}_{\mathcal{S}}\mathbb{P}\big(Y\not\in \mathcal{S}(X)\big)+\lambda\mathbb{E}\big[|S(X)|\big],\]

o√π on affiche la d√©pendance en \(\lambda\). Il se trouve que ce probl√®me poss√®de une forme close que nous donnons imm√©diatement et qui revient √† seuiller les probabilit√© :

\[\mathcal{S}_\lambda^\star(x)=\{k\in\{1, \ldots, C\}:\¬†\eta_k(x)\geq \lambda\}.\]

Il s‚Äôagit maintenant de choisir une valeur correcte de \(\lambda\) en se rappelant que notre point de d√©part est notre souhait de retourner en moyenne \(K\) √©l√©ments. Notons :

\[G(t)=\sum_{k=1}^C\mathbb{P}(\eta_k(X)\geq t).\]

C‚Äôest l‚Äôesp√©rance de la taille de notre ensemble pr√©dit si on seuil √† \(t\). Consid√©rons son inverge g√©n√©ralis√©e, not√©e \(G^{-1}(k)\). Notre classifieur de Bayes devient via ce dernier :

\[\mathcal{S}_\bar{k}^\star(x)=\{k\in\{1, \ldots, C\}:\¬†\eta_k(x)\geq G^{-1}(\bar{k})\}.\]

B. En pratique¬∂

C‚Äôest bon, nous avons toutes les billes :

  • On entra√Æne notre mod√®le sur un jeu d‚Äôapprentissage pour estimer \(\hat{\eta}\),

  • On estime \(G^{-1}(\bar{k})\) sur un jeu de validation.
